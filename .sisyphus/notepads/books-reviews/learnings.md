## [2026-03-29] Task 2 Learnings: better-auth + Prisma downgrade
- Prisma v7 (installed by default) is INCOMPATIBLE with this setup ‚Äî downgraded to v5.22.0
- Prisma v7 requires `adapter` or `accelerateUrl` in PrismaClient constructor ‚Äî no `url` in schema
- Prisma v5 is the stable version ‚Äî uses `url = env("DATABASE_URL")` in schema.prisma datasource
- prisma.config.ts was generated by v7 init ‚Äî NOT needed in v5, MUST DELETE for build to pass
- better-auth package: `better-auth/adapters/prisma` exports `prismaAdapter`
- better-auth Next.js integration: `better-auth/next-js` exports `toNextJsHandler`
- better-auth client: `better-auth/client` exports `createAuthClient`
- BETTER_AUTH_URL env var required for better-auth to determine base URL (set to NEXT_PUBLIC_APP_URL value)
- Build warning: "Base URL could not be determined" ‚Äî fixed by setting BETTER_AUTH_URL env var
- Schema must NOT have forward references to models not yet defined (T3 models must wait)
- Prisma v5 generate works fine with v5 schema format (url in datasource)
- pnpm approve-builds needed for @prisma/client, @prisma/engines, prisma packages
- **Base UI Button Composition**: When using the new base-nova shadcn style that uses `@base-ui/react`, the `Button` component uses a `render` prop instead of `@radix-ui/react-slot`'s `asChild`. Example: `<Button render={<Link href="/search" />}>Search</Button>`.
- **Client module imports in RSC**: `buttonVariants` exported from `components/ui/button.tsx` (a `"use client"` module) cannot be used in a Server Component. Wrap with a client component or use the standard `Button` with `render` prop.
